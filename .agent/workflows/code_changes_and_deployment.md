# 에이전트 소스 코드 변경 및 배포 지침

에이전트는 코드 수정 후 다음 원칙을 반드시 준수해야 합니다.

## 1. 로컬 환경 우선 적용
- 요청받은 소스 코드 변경 사항은 **오직 로컬(Local) 환경**에만 적용합니다.
- 로컬 적용 후 `npm run dev` 등을 통해 정상 작동 여부 및 에러를 확인합니다.
- `git add` 및 `git commit`을 통해 로컬 Git에 기록은 남길 수 있으나, **절대로 원격 저장소(GitHub)로 자동 `git push` 하거나 프로덕션 환경(Firebase 등)에 자동 배포하지 않습니다.**

## 2. 사용자 검토 완료 대기
- 코드 수정 작업이 끝나면, 어떤 사항이 반영되었는지 사용자에게 설명하고 화면에서 직접 확인해볼 것을 안내합니다.
- 사용자에게 배포 여부를 물어본 뒤 승인을 기다립니다.

## 3. 사용자 승인 후 깃허브 푸시 및 배포
- 사용자가 코드를 확인하고 명시적으로 **"승인", "배포해줘", "푸시해줘", "완료"** 등의 승인 의사를 밝힌 경우에만 다음 작업을 수행합니다.
  1. 깃허브 원격 저장소 푸시 (`git push`)
  2. 프로덕션 빌드 작업 (`npm run build`)
  3. 실서버 배포 (`npx firebase deploy` 등)

**주의사항:** 사용자의 확실한 배포/푸시 승인이 있기 전까지는 임의로 운영 서버나 원격 저장소에 변경 사항을 올리지 마십시오.
