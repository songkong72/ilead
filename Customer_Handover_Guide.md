# 엘이에이디 홈페이지 소스 이행 및 외부 서비스 연동 가이드

본 문서는 엘이에이디 홈페이지의 소스 코드 인수인계 절차와 카카오 서비스(로그인, 상담) 연동을 위해 고객사에서 준비해야 할 사항을 정리한 가이드입니다.

---

## 1. 소스 이행 전 사전 준비 항목

고객사에서 시스템 소유권을 이전받기 위해 아래 계정들이 사전에 준비되어야 합니다.

| 구분 | 필수 준비 항목 | 목적 |
| :--- | :--- | :--- |
| **코드 관리** | **GitHub 계정** | 소스 코드 저장소 권한 이양 및 자동 배포(CI/CD) 설정 |
| **인프라** | **Google 계정 (Firebase)** | 호스팅 유지, 데이터베이스 관리, 사용자 인증 관리 |
| **이미지 서버** | **ImgBB 계정** | 관리자 모드에서 실시간 이미지 업로드 및 저장 (API 키 발급) |
| **도메인** | **관리 도메인** | 가비아 등에서 구매한 도메인 연결 정보 준비 |

---

## 2. 외부 서비스 연동 준비 (카카오)

### 2.1 카카오톡 상담 채널 설정
홈페이지 우측 하단의 상담 버튼을 실제 고객사 채널로 연결하는 단계입니다.

1. **채널 생성**: [카카오톡 채널 관리자센터](https://center-pf.kakao.com/)에서 새 채널 생성.
2. **공개 설정**: [프로필 설정] 메뉴에서 '채널 홈' 공개를 **ON**으로 변경.
3. **채팅 활성화**: [상담 설정] 메뉴에서 '채널 채팅'을 **사용**으로 설정.
4. **링크 확인**: 내 채널의 고유 링크(`https://pf.kakao.com/_xxxxxx`)를 복사.
5. **홈페이지 반영**: 홈페이지 관리자 모드 로그인 후, 문의 버튼의 설정(톱니바퀴) 아이콘을 눌러 링크 입력.

### 2.2 카카오 로그인 연동 설정 (API)
홈페이지 회원가입 및 로그인을 카카오 계정으로 연동하기 위한 권한 획득 단계입니다.

1. **앱 등록**: [카카오 디벨로퍼스](https://developers.kakao.com/)에서 애플리케이션 추가.
2. **플랫폼 등록**: [플랫폼] 메뉴에서 본 사이트의 도메인 주소 등록.
3. **로그인 활성화**: [카카오 로그인] 메뉴에서 상태를 **ON**으로 변경.
4. **Redirect URI 설정**: Firebase 인증 핸들러 주소 등록 (개발자 가이드 필요).
5. **키 전달**: [요약 정보]의 **REST API 키**를 확인하여 개발자에게 전달.

## 3. 상세 인수인계 절차 (Handover)

1. **저장소 이전 (GitHub)**: 개발자가 관리하던 소스 저장소를 고객사 GitHub 계정으로 Transfer.
2. **Firebase 프로젝트 이관**: 
   - 고객사가 생성한 Firebase 프로젝트에 개발자를 초대.
   - 데이터베이스(Firestore) 및 호스팅 설정을 고객사 프로젝트로 이전.
3. **환경 변수(.env) 업데이트**: 
   - 고객사가 발급받은 Firebase API 키, ImgBB 키, 카카오 API 키로 시스템 환경 설정 변경.
4. **CI/CD 연결**: GitHub Actions를 통해 코드를 수정하고 Push하면 자동으로 사이트가 업데이트되도록 최종 연결.
5. **운영 교육**: 관리자 모드 사용법(이미지 교체, 텍스트 수정, 공지사항 관리) 안내.

---

## 4. 보안 및 환경 변수 관리 (CRITICAL)

운영 시 보안을 위해 아래의 환경 변수 관리 원칙을 반드시 준수해야 합니다.

### 4.1 .env 파일 관리
- **절대 금지**: `.env` 파일을 GitHub 저장소에 Push하지 마십시오. (이미 `.gitignore`에 포함되어 있어야 합니다.)
- **로컬 전용**: `.env`는 오직 개인의 로컬 개발 환경에서만 사용합니다.

### 4.2 운영 환경 키 설정 (GitHub Secrets)
실제 배포된 사이트에서 사용할 키값은 **GitHub 저장소의 설정**에서 관리합니다.
1. GitHub 저장소 > **Settings** > **Secrets and variables** > **Actions** 메뉴로 이동.
2. **New repository secret** 버튼 클릭.
3. 아래 항목들을 각각 등록 (예: `VITE_FIREBASE_API_KEY` 등).
4. 자동 배포 스크립트(`firebase-deploy.yml`)가 이 값을 참조하여 안전하게 빌드합니다.

### 4.3 Firebase 보안 규칙
- 키값이 노출되더라도 데이터가 안전하도록 **Firebase Firestore Security Rules**를 반드시 설정해야 합니다. (관리자 외 쓰기 권한 제한 등)
